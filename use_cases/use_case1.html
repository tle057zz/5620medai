Overview

Request Insurance Quote Obtain tailored health insurance quote Business User has entered health data, past medical records, and income details into the system AI provides insurance options tailored to user’s health, history, and financial condition No insurance quote generated or irrelevant/unusable quotes provided User is logged in and selects Request Insurance Quote

Main Scenario (Happy Path)

  1. User logs into the healthcare system.
  2. User navigates to the Insurance section and selects Request Insurance Quote.
  3. System prompts the user to enter/update health data, past medical history, and income details.
  4. User submits the required information.
  5. AI engine processes the input and searches available insurance options.
  6. AI generates tailored insurance quotes based on health conditions, past records, and income.
  7. System displays recommended insurance quotes with details (coverage, cost, provider).
  8. User reviews quotes and saves/ downloads them for reference.

Nested Paths (Sub–Use Cases)

1) Login to Healthcare System — “Login”

  1. Ask for credentials (email/ID + password).
  2. User enters credentials.
  3. Validate credentials (hash check, account status).
  4. If enabled, prompt & verify MFA (OTP/biometrics).
  5. Create session & set auth token.
  6. Redirect user to dashboard.

2) Open Insurance Quote Module — “Navigate to Insurance”

  1. User selects Insurance from navigation.
  2. System loads Insurance home (policies, quotes, eligibility).
  3. Check profile completeness flags (health, records, income).
  4. Display Request Insurance Quote action.

3) Collect Data — “Enter/Update Health, Records, Income”

  1. Show structured form (health conditions, meds, hospitalizations, income).
  2. Pre‑fill from EHR/previous entries; label data freshness.
  3. Ask for consents to use/share data with insurers.
  4. User edits/attaches documents (e.g., discharge letter).
  5. Validate fields (required, ranges, cross‑field logic).
  6. Save draft to profile.

4) Submit Request — “Submit Quote Request”

  1. User clicks Submit.
  2. Package payload (PHI minimization, pseudonymization).
  3. Assign request ID and enqueue for processing.
  4. Return acknowledgement and status badge (“Processing…”).

5) Process & Search Options — “Orchestrate Quote Retrieval”

  1. Normalize input (terminology mapping: ICD/ATC, income brackets).
  2. Run risk stratification & eligibility rules.
  3. Build search criteria (coverage needs, exclusions, budget).
  4. Call insurer product catalogs/APIs (with retries/backoff).
  5. Collect candidate products & premiums.

6) Generate Tailored Quotes — “Compose Recommendations”

  1. Score options (fit vs. conditions, provider network, OOP limits).
  2. Rank by total cost of ownership (premium + expected OOP).
  3. Compute explanations (why recommended / why excluded).
  4. Produce recommendation set (JSON) with rationale & confidence.

7) Display Quotes — “Present Results”

  1. Render quote cards (premium, deductible, copay, exclusions).
  2. Show filters/sort (price, coverage depth, network).
  3. Provide View details, Compare, and Download PDF.
  4. Log presentation for audit & caching.

8) Review & Save/Download — “User Acts on Quotes”

  1. User compares shortlisted plans.
  2. User saves favorites to profile.
  3. User downloads/prints PDF summary.
  4. (Optional) Share with doctor / request human advisor follow‑up.
  5. Persist selection & update timeline/history.

Extension Paths

1) Doctor Involvement for Validation

  • Trigger: After quotes are generated (Step 6), user enables “Doctor review required.”
  1. System routes generated quotes to the assigned doctor.
  2. Doctor reviews AI recommendations.
  3. Doctor adds notes or overrides AI ranking if needed.
  4. Updated, validated quotes are shown to user.

Outcome: User receives medically validated quotes rather than raw AI suggestions. Next: Continue to normal path Step 7.

2) User Requests Cost Breakdown

  • Trigger: While viewing quotes (Step 7), user clicks “See detailed cost breakdown.”
  1. System expands quote details: premium, deductibles, co‑pay, out‑of‑pocket maximums.
  2. AI provides a projected cost simulation (e.g., “Estimated yearly expense if hospitalized twice”).
  3. User can compare this breakdown across multiple plans.

Outcome: User gains deeper cost insight before deciding. Next: Continue to normal path Step 8.

Failure Paths

A) Derived from Step 3 — Data Collection Errors

  • Failure 3a.1: User skips mandatory fields (e.g., income). Response: Show error and request correction.
  • Failure 3a.2: Invalid or inconsistent data (e.g., negative income). Response: Display validation error and block submission.
  • Failure 3a.3: System cannot retrieve past medical records. Response: Notify user; allow retry or proceed with partial data.
  • Failure 3a.4: Session timeout before completion. Response: Redirect to login; prompt to re‑enter information.

B) Derived from Step 6 — AI/Integration Errors

  • Failure 6a.1: AI cannot process due to corrupted/incomplete input. Response: Show error; prompt to fix data.
  • Failure 6a.2: No matching insurance products found. Response: Inform user; suggest contacting human advisor.
  • Failure 6a.3: External insurer APIs unavailable. Response: Show partial results with warning; ask to try later.
  • Failure 6a.4: Internal system error (timeout/overload). Response: Log error; notify temporary issue; allow retry.
